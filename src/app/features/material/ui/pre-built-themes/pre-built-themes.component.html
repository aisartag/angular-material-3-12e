Utilizzo di un tema predefinito (pre-built)

In Material 3 i temi predefiniti sono stati ridotti ai quattro sotto elencati:

Tema {{ element.css }} Light o Dark {{ element.lightOrdark }} Color primary {{ element.colorPrimary }} Color tertiary {{ element.colorTertiary }}

Stato dell' applicazione dopo l'installazione di un pre-built

E' intervenuta nei seguenti file

  • src/app.config.ts

    Dando un'occhiata a questo file nella configurazione osserviamo la lista providers. Contien le funzionalità di base prima inserite nei moduli. E qui la nostra installazione di material ha inserito il provider di animazione provideAnimationsAsync()

  • src/index.html

    In index.html l'installazione di material ha inserito i link alle font googlr Roboto. Nel body è stata aggiunta la class mat-typography secondo la nostra scelta dell'uso della tipografia globalmente

  • angular.json

    Ad esempio dopo aver scelto il tema azure-blue

    
          "styles": [
           "@angular/material/prebuilt-themes/azure-blue.css",
           // other styles ],

    Questo file include il CSS per ogni componente nella libreria. Per includere solo il CSS per un sottoinsieme di componenti, devi usare la Sass API descritta in seguito.


  • src/style.scss 
    
  html, body { height: 100%; }
  body { margin: 0; font-family: Roboto, "Helvetica Neue", sans-serif; }

L'installazione di un tema predefinito è pronta. Conosciamo a questo punto le operazioni per ripristinare un tema predefinito ( consogliabile per chi non vuole cimentarsi con sass!).
